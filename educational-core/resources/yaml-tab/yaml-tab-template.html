YAML format was introduced to make course configuration easier and transparent. Now it's possible to change the course structure by editing *.yaml files. Here's a description of the format for each course item.

Course

course-info.yaml describes the structure of a course:
Name Description Value
title Title of the course String
language Languages used for course materials Any of supported languages, e.g. English, German, French, Spanish, Russian, etc.
summary Summary describing what the course is about String
programming_language Programming language of the course Any of supported programming languages, e.g. Java, Kotlin, Scala, JavaScript, Rust
programming_language_version Optional version of programming language Number or String
environment Additional information about the course environment For example, Kotlin courses are supported for Android environment
content Directory names of lessons or sections included in the course List of directory names
solutions_hidden If solutions are available for learners to peek and compare with their answers true or false
Default: true

Section

section-info.yaml describes the structure of a section:
Name Description Value
custom_name Custom name for the section if it differs from its directory name String
content directory names of the lessons included in the section List of directory names

Lesson

lesson-info.yaml describes the structure of a lesson:
Name Description Value
type Lesson type lesson
framework
custom_name Custom name for the lesson if it differs from its directory name String
content Directory names of tasks included in the section List of directory names

Task

task-info.yaml describes the structure of a task:
Name Description Value
type Task type edu theory output ide code choice
custom_name Custom name for the task if it differs from its directory name String
files Files included in the task List of files in format described below
feedback_link Custom link for learners to leave their feedback about the task E.g., Google Forms link
is_multiple_choice If choice task allows multiple correct answers true false
message_correct choice Custom message to be shown to learner in case of correct solution E.g., “Great job!” or “Fantastic!”
message_incorrect choice Custom message to be shown to learner in case of incorrect solution E.g., “Review how to define a function to solve this task”
options choice task answer variants List of options. Option has two parameters: text and is_correct

file format

Name Description Value
name File path relative to the directory of the task String
visible If the file is shown to learner or not true false
placeholders Answer placeholders List of placeholders in format described below

placeholder structure

Name Description Value
offset Placeholder start offset Number
length Placeholder length Number
placeholder_text Text which will be shown to learner in place of the placeholder String
dependency Dependency on another placeholder placeholder dependency in format described below

placeholder dependency structure

Name Description Value
section Section folder name if placeholder dependency is inside a section String
lesson Lesson directory name String
task Task directory name String
file File path relative to the directory of the task String
placeholder Placeholder index starting from 0 Number
is_visible If placeholder is drawn in place of dependency true or false
Default: true
